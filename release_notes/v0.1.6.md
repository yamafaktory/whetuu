## Upgrading from 0.1.5 or older

**Move the binary.** whetuu now installs to `~/.local/bin`, the path the XDG
Base Directory spec names for user executables. Older versions installed to
`~/.whetuu/bin` and put that at the front of your `PATH`, so the old binary
keeps winning until you clear it out:

```sh
rm -rf ~/.whetuu
```

Then drop the `export PATH="$HOME/.whetuu/bin:$PATH"` line (or the
`fish_add_path` equivalent) from your shell config and open a new shell. Your
history is not in `~/.whetuu`, so nothing is lost. Re-running the installer
handles the rest, and tells you when it finds the old directory.

**`whetuu prompt` is now `whetuu render`.** The old name is gone, not
deprecated.

Nothing to do if your shell config uses `eval "$(whetuu init bash)"` or
`whetuu init fish | source`, which is what the installer writes and what the
README documents. Those regenerate the integration script on every shell start,
so they pick up the new name by themselves.

If instead you copied the generated script into your config, it still calls
`whetuu prompt`. You will get an empty status line and the command list printed
before every command until you replace that block with the one line above. Your
shell keeps working throughout. `whetuu init <shell>` prints the line and the
file it belongs in.

## What changed

**Installs to `~/.local/bin`.** The XDG Base Directory spec names it for user
executables, and most systems already have it on `PATH`. whetuu now creates no
directory of its own in `$HOME`. The installer adds a `PATH` line only when it
has to, so the shell config usually gains one line instead of two.

**A status line, not a prompt.** whetuu draws the line above your cursor and it
binds the up arrow to a history picker. Calling it a prompt named only the first
half. Nothing it does has changed, and your shell's own history file is still
never read or written.

**Uninstalling is documented.** The binary, the history store, the version cache
and the shell config block. `whetuu paths` prints the two data locations, which
move with `$XDG_DATA_HOME` and `$XDG_CACHE_HOME`.

**The install docs lead with the download.** Fetching the tarball and checking
it against `SHA256SUMS` is now the first route rather than a collapsed section.
The installer does the same work and verifies nothing you could not verify
yourself.

**The Nerd Font note is accurate.** Most terminal setups already run one, so try
whetuu before installing a font. The old wording read as a hard prerequisite.

**Fixed:** the installer printed the default install directory in the shell
config lines even when `WHETUU_INSTALL_DIR` had put the binary elsewhere.
